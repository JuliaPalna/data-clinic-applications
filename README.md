### API:
- POST   /api/login   - авторизация, получение токена
- POST   /api/logout  - выход, удаление токена
- POST   /api/tickets - получение массива заявок 

получение странииц:
- GET   /login    - авторизация
- GET   /tickets  - список заявок (проверка аутентификации)


# Задача:

К вам обратился заказчик — частная клиника, которая нуждается в системе по сбору заявок. Требуется страница с формой, куда можно заполнить информацию о себе и своей проблеме. Данные отправляются в клинику, и в ближайшее время с пациентом связывается оператор. Он предлагает врача по описанной проблеме и согласует дату и время приема.

## Функциональные требования:

- В приложении должна быть возможность оставить заявку (отправить форму с данными).
- Для полей ввода должна быть соответствующая валидация.
- Отправленные данные должны появиться в таблице заявок.
- Доступ к странице с таблицей заявок должен быть только у авторизованных пользователей по отдельному URL. Данные для входа выдает клиника, система регистрации не предусмотрена.
## Технические требования:
- Frontend —  React.
- Backend — Node.js, Express.js, MongoDB.
- Заявки хранятся в MongoDB
## Все страницы
Дизайн на усмотрение разработчика (но должен быть аккуратный, приятный на вид).

## 1. Форма
- Форма, содержащая следующие поля ввода (минимум 3):
-  ФИО (обязательное).
-  Номер телефона (обязательное, желательно добавить маску ввода).
-  Опишите вашу проблему (необязательное, область для ввода текста).
- Кнопка “Отправить” (после отправки блокируется).
- После успешной отправки формы появляется сообщение об успешной отправке, поля для ввода очищаются и кнопка становится активной.

## 2. Страница входа
Эта страница входа только для операторов (сотрудников клиники)

Содержаие:

- Форма, содержащая следующие поля ввода:
- Email
- Пароль
- Кнопка “Войти”

## 3. Страница с таблицей заявок
- Таблица с данными. Данные из заявки распределяются по колонкам таблицы.
- Должна быть колонка с датой и временем отправки (определяется автоматически при отправке заявки).
